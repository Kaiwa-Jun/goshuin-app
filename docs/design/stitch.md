# Stitchデザインデータ

## 概要

UIデザインは [Stitch](https://stitch.withgoogle.com/) で作成しています。デザインデータはStitch MCPを使用して実装コード側に直接取り込みます。

## デザインデータの役割分担

| 場所 | 内容 |
|---|---|
| `docs/design/ui-design.md` | 画面仕様・遷移フロー・UX設計意図（設計ドキュメント） |
| `src/` (実装コード側) | Stitch MCPで取り込んだコンポーネント・スタイル |

## 取り込み方針

- **設計意図**はdocsに、**実装詳細**はコード側に配置
- Stitch MCPで取り込む際は、実装コードのディレクトリ（`src/components/` など）に直接配置
- 取り込んだコンポーネントは必要に応じてプロジェクトの規約に合わせて調整

## 取り込み手順

1. Stitch MCPを使用してデザインデータを取得
2. 取得したコンポーネントを `src/components/` 配下に配置
3. プロジェクトの命名規則・ディレクトリ構造に合わせて調整
4. 必要に応じてスタイルを `src/styles/` に整理

## 注意事項

- 取り込んだコンポーネントは、Expoプロジェクトの技術スタック（React Native + TypeScript）に適合させる
- Rive対応予定のコンポーネント（登録完了演出、バッジ、FABボタン等）は、後から差し替えやすいよう独立したコンポーネントとして切り出す
