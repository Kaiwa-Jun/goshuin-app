# Issue別詳細設計ドキュメント

このディレクトリには、Issue単位でClaude Codeとやりとりして作成した詳細設計ドキュメントを格納します。

## 運用フロー

1. **Issue作成**: GitHub Issue を作成し、実装対象の機能やタスクを定義する
2. **詳細設計**: Claude Code とやりとりして詳細設計を立てる
3. **ドキュメント作成**: 詳細設計の結果をこのディレクトリ内にmdファイルとして保存する
4. **実装**: `/clear` した後、作成した詳細設計mdファイルを参照して実装を進める

## ファイル命名規則

```
docs/issues/
├── README.md                          # このファイル
├── issue-001-project-setup.md         # 例: プロジェクト初期セットアップ
├── issue-002-supabase-setup.md        # 例: Supabase環境構築
├── issue-003-map-screen.md            # 例: 地図画面の実装
└── ...
```

**命名フォーマット**: `issue-{番号}-{概要}.md`

- 番号: GitHub Issueの番号に対応（3桁ゼロ埋め）
- 概要: ハイフン区切りの英語で簡潔に

## ドキュメントテンプレート

```markdown
# Issue #{番号}: {タイトル}

## 概要
{この Issue で実装する内容の概要}

## 関連ドキュメント
- [要件定義](../product/requirements.md)
- [技術設計](../technical/tech-design.md)

## 詳細設計

### 対象ファイル
{変更・追加するファイルの一覧}

### 実装方針
{実装のアプローチ、選択した方法とその理由}

### データ構造
{必要に応じてテーブル定義や型定義}

### API / エンドポイント
{必要に応じてAPI設計}

### 画面仕様
{必要に応じてUI仕様の補足}

## テスト方針
{テストの方針や確認項目}

## 注意事項
{実装時に気をつけるべきこと}
```
